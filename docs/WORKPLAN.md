# БескромныйAI — План работы

## Структура проекта (создана)

```
БескромныйAI/
├── bot/                    # Telegram-бот
│   ├── handlers/           # Обработчики команд (/post, /rewrite, ...)
│   ├── services/           # Бизнес-логика бота
│   └── main.py             # Entry point
├── core/                   # Ядро системы
│   ├── rag/                # ChromaDB: store, retrieval, embeddings
│   ├── voice/              # Voice Profile: промпты, регистры
│   ├── feedback/           # SQLite фидбек-петля
│   └── config.py           # Конфигурация (.env)
├── scripts/                # Скрипты загрузки данных
├── web/                    # Next.js + Tailwind витрина
│   └── src/app/            # App Router
├── data/                   # Локальные данные (не в git)
│   ├── chroma/             # ChromaDB persistence
│   ├── transcripts/        # Аудио и транскрипты
│   └── exports/            # Экспорты данных
├── docs/                   # Документация
├── CLAUDE.md               # Контекст для Claude
├── pyproject.toml          # Python зависимости
└── .env.example            # Шаблон переменных окружения
```

## ФАЗА 1 — MVP: бот который знает голос (Недели 1–2)

### 1.1 RAG-ядро (Блок A)
- [ ] `scripts/load_channel_archive.py` — выгрузка постов канала через Bot API
- [ ] Чанкинг постов + генерация embeddings (text-embedding-3-small)
- [ ] Загрузка в ChromaDB с метаданными (register, date, topic_tags, type)
- [ ] `core/rag/retrieval.py` — semantic search с фильтрами по регистру и дате
- [ ] Временное взвешивание: x1.5 для постов < 12 месяцев

### 1.2 Voice Engine (Блок 0 в коде)
- [ ] `core/voice/profile.py` — загрузка Voice Profile из Google Docs
- [ ] `core/voice/prompts.py` — шаблоны промптов по командам (/post, /rewrite, /comment)
- [ ] Интеграция RAG context + Voice Profile + Claude API

### 1.3 Telegram-бот (три команды)
- [ ] Реализация `/post [тема]` — RAG retrieval + генерация + 3 варианта заголовка
- [ ] Реализация `/rewrite` + текст — адаптация в голосе
- [ ] Реализация `/comment` + текст — ответ на комментарий
- [ ] `/check [тезис]` — semantic search по архиву на повторы

### 1.4 Фидбек-петля (Блок D)
- [ ] Inline-кнопки после каждой генерации: ✓ Использовал / ✎ Отредактировал / ✗ Не подошло
- [ ] При «Отредактировал» — запрос финальной версии, сохранение пары
- [ ] SQLite запись (уже создана схема в `core/feedback/db.py`)

**Результат Фазы 1:** бот в Telegram, который пишет посты в голосе Димы. Реальная польза с первого дня.
