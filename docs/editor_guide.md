# Инструкция для редактора — БескромныйAI

## Что это

Telegram-бот, который генерирует контент в голосе Дмитрия Бескромного. Бот обучен на 6000+ постов канала, видео-транскриптах, подкастах и статьях. Он не заменяет редактора — он даёт черновик, с которым ты работаешь.

## Как начать

1. Открой бот в Telegram (ссылку даст Дмитрий)
2. Напиши `/start` — увидишь список команд
3. Если бот ответит «Нет доступа» — скинь свой Telegram ID Дмитрию (он будет в сообщении)

## Основные команды

### Генерация контента

| Команда | Что делает | Пример |
|---------|-----------|--------|
| `/post [тема]` | Пост для канала (800-2000 зн.), 3 варианта заголовка | `/post цифровой суверенитет` |
| `/rewrite [текст]` | Переписывает любой текст в голосе Димы | `/rewrite Искусственный интеллект меняет бизнес...` |
| `/comment [текст]` | 3 варианта ответа на комментарий | `/comment Считаю что AI убьёт рабочие места` |
| `/article [тема]` | Длинная статья (3000-5000 зн.), стиль Forbes/РБК | `/article будущее медиа после AI` |
| `/research [тема]` | Синтез темы по архиву Димы | `/research медиабизнес в России` |

Каждая генерация автоматически попадает в **очередь контента** с номером (#1, #2, ...).

### Работа с очередью

| Команда | Что делает |
|---------|-----------|
| `/queue` | Показать активные черновики |
| `/queue edited` | Показать только отредактированные |
| `/view [id]` | Посмотреть полный текст черновика |
| `/edit_draft [id]` | Начать редактирование — следующее сообщение сохранится как правка |
| `/search [запрос]` | Поиск по архиву — что Дима уже писал на тему |
| `/recent` | Последние 10 постов канала |
| `/check [тезис]` | Проверка — не было ли такого уже |

### Только для Дмитрия

| Команда | Что делает |
|---------|-----------|
| `/approve [id]` | Одобрить черновик |
| `/reject [id] [причина]` | Отклонить с комментарием |

## Рабочий процесс

### Шаг 1: Генерация

Пишешь `/post цифровой суверенитет` — бот выдаёт черновик и присваивает ему номер в очереди.

### Шаг 2: Проверка повторов

Перед публикацией проверь: `/check [основной тезис поста]`. Бот покажет, писал ли Дима что-то похожее.

### Шаг 3: Исследование контекста

Если нужно больше контекста — `/search [тема]` покажет релевантные посты из архива. Это поможет понять, какую позицию Дима занимает по теме.

### Шаг 4: Редактирование

1. Посмотри черновик: `/view 15`
2. Начни редактирование: `/edit_draft 15`
3. Отправь отредактированную версию следующим сообщением
4. Статус изменится на «edited» — ждёт одобрения Дмитрия

### Шаг 5: Фидбек

После каждой генерации бот показывает три кнопки:
- **"Использовал"** — взял как есть
- **"Отредактировал"** — бот попросит отправить финальную версию
- **"Не подошло"** — не подошло

**Важно**: нажимай кнопки честно! Это обучает систему. Если ты всегда правишь — нажимай "Отредактировал" и отправляй финальный текст. Эти пары "оригинал → правка" — самое ценное для улучшения бота.

## Голос Димы — ключевые правила

Бот знает эти правила, но тебе полезно их помнить при редактуре:

- **Тире** — длинное (—), не дефис
- **Без эмодзи** в теле текста
- **Без восклицательных знаков**
- **Первая строка** — парадоксальная или конкретная (не банальная)
- **Концовка** — панчлайн, вопрос или тишина. Никогда — резюме
- **Без выдуманных цифр** и источников
- Команда `/style` покажет полный Voice Profile

## Частые вопросы

**Бот долго думает**
Генерация занимает 30-120 секунд — это нормально. Бот показывает индикатор «печатает...».

**Бот выдал ерунду**
Попробуй переформулировать тему. Чем конкретнее запрос — тем лучше результат. «Пост про AI» — плохо. «Почему российский бизнес недооценивает AI-агентов» — хорошо.

**Как посмотреть статистику**
`/stats` — количество документов в базе, статистика по фидбеку.

**Как узнать свой Telegram ID**
Напиши боту — если нет доступа, он покажет твой ID в сообщении.
